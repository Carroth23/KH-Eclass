유스케이스다이어그램 용어
- 시스템 : 만들고자 하는 프로그램
- 액터 : 시스템과 상호작용을 하는 사람(졸라맨)
- 유스케이스 : 사용자입장에서 바라본 시스템의 기능 
-- 관계는 4종류가 있다.
- 연관관계(Association) 실선 (액터가 유스케이스를 사용하는것)
- 포함관계(Include) 화살표 점선(유스케이스가 다른 유스케이스의 실행을 전제로 할때)
- 확장관계(Extend) (특정 조건에 따라? 유스케이스간에 사용되는)
- 일반화 관계(Generalization) 상속받는거인듯

클래스간의 결합도
- 서로 다른 모듈간에 상호 의존정도.

코드 응집도
- 모듈이 독립적인 기능을 수행하는지.
음.. 메서드가 모듈이라고 하면 하나의 메서드는 하나의 기능만을 수행하는지?

모듈(모듈화는 뭔지)
- 모듈화는 소프트웨어를 각 기능별로 나누는것.
각각 기능별로 나눠진 모듈화의 결과를 모듈이라고 함 ㅎㅎ

공통모듈
- 여러 프로그램에서 공통으로 사용할 수 있는 모듈
static 같은 개념인듯

서버 가용성
- 서비스가 다운되지 않고 정상적으로 유지된 시간

이중화
- 서버나 시스템에 장애발생했을때, 시스템 기능을 계속 유지할 수 있도록 예비장치를 배치해 운용하는것.

폴트톨러런스
- 시스템의 모듈중 어느 하나에 장애가 발생해도 시스템 운영에 전혀 지장없게 설계된 컴퓨터시스템
